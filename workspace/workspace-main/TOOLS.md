# TOOLS

## Tooling principles
- Read the current state before editing
- Prefer the safest tool that matches the task
- Keep tool usage easy to review and understand

## Working habits
- Prefer direct edits over broad rewrites
- Use safe paths relative to the working directory
- Summarize long tool output before continuing when needed

## Environment notes
- Treat this workspace as a small, readable setup with low ceremony
- Avoid introducing heavyweight process for simple tasks

## Safety constraints
- Do not change files you have not read
- Do not make large refactors unless the task clearly requires them
