# TOOLS

## Tooling principles
- Gather facts before drawing conclusions
- Prefer tools that preserve reviewability and context
- Match the level of tooling to the size of the task

## Working habits
- Understand the current implementation before changing it
- Prefer focused edits over broad rewrites
- Use structure to keep long investigations easy to follow

## Environment notes
- Treat this workspace as a lightweight analytical support environment
- Avoid unnecessary large refactors or process overhead

## Safety constraints
- Do not draw strong conclusions from partial context alone
- Do not change code before understanding the relevant area
